## 氣泡排序法

### 氣泡排序法 介紹

從第一個元素開始，比較相鄰元素大小，如果順序有誤，則對調再進行下一個元素的比較。掃描過一次後就可以確保最後一個元素是位於正確的位置。接著再進行第二次掃描，直到所有元素完成排序為止。

### 氣泡排序法 參考程式碼

```cpp
void bubble_sort(int a[], int n)
{
    for (int k=n-1; k > 0; k--) {
        for (int i=0; i < k; i++) {
            if (a[i] > a[i + 1]) {
                // swap a[i] and a[i+1]
                int t = a[i];
                a[i] = a[i + 1];
                a[i + 1] = t;
            }
        }
    }
}
```

### 氣泡排序法 複雜度分析

時間複雜度: $ O(n^2) $

* 迴圈跑 n-1 次
* 迴圈每跑一次，分別進行進行 n-1, n-2, … 1 次比較
* 共比較 1 + 2 + … + (n-1)
= $ n(n-1) / 2 $
= $ n^2/2 - n/2 $ 次
* 當 n 很大時，主要看最高次方的項次即可 ≈ $ n^2/2 $ 次
* 複雜度為多項式等級，一般不管係數，將其計為 $ O(n^2) $

空間複雜度: $O(1)$

不需要額外的陣列去存資料，因此為 $O(1)$

## 選擇排序法

### 選擇排序法 介紹

將資料分成已排序及未排序兩部分，依序在未排序中找最小值，加入到已排序部分的末端。按照這個方式，可以在第一輪中找到最小的，第二輪找到第二小的。一般會先設未排序陣列的第一個數字是目前的最小值，然後再往後一個一個比大小，如果後面的數字比目前的最小值小，那就將目前的最小值換為這個數，之後只要重複操作即可。

### 選擇排序法 參考程式碼

```cpp
void selection_sort(int a[], int n)
{
    for (int k=0; k < n-1; k++) {
        int i = k;
        for (int j=i+1; j<n; j++) {
            if (a[i] > a[j]) i = j;
        }
        // swap a[k] and a[i]
        int t = a[k];
        a[k] = a[i];
        a[i] = t;
    }
}
```

### 選擇排序法 複雜度分析

時間複雜度: $ O(n^2) $

* 迴圈由 k 控制，k = 0, 1, 2,  … n-2
* 比較 n-1-k 次
* 總共比較次數 = 1 + 2 + … + (n-1)
= $ n(n-1) / 2 $
* 交換次數 = n-1
* 複雜度為 $ O(n^2) $
* 每回合比完都要互換，共互換 n-1 次

空間複雜度: $O(1)$

不需要額外的陣列去存資料，因此為 $O(1)$

## 插入排序法

### 插入排序法 介紹

類似平常玩撲克牌時，將其由小到大排序的過程。可以將第 $ i $ 張牌加入前 $ i-1 $ 張以排序過的牌，那就可以獲得 $ i $ 張排序過的牌組了。

### 插入排序法 參考程式碼

```cpp
void insertion_sort(int a[], int n)
{
    int sorted = 0;
    for (int tosort=1; tosort<n; tosort++) {
        int save = a[tosort];
        int pos = tosort;
        while (pos>0 && a[pos-1] > save) {
            a[pos] = a[pos-1];
            pos--;
        }
        a[pos] = save;
    }
}
```

### 插入排序法 複雜度分析

時間複雜度: $ O(n) $ ~ $ O(n^2) $

* 迴圈由 tosort 控制，tosort = 1, 2, … n-1
* 執行次數不定，介於 1~tosort 之間
* 最差情況要比 2 * tosort (比較加移動)
* 依最差情況計算
總次數 = 2 * (1 +2 + … + n)
 = n (n+1)
* 最好情況只比 1 次
依最好情況計算
總次數 = 2 * n

空間複雜度: $O(1)$

不需要額外的陣列去存資料，因此為 $O(1)$

## 合併排序法

### 合併排序法 介紹

分治法(Divide-and-conquer):
把一個複雜的問題分成兩個或更多的相同或相似的子問題，直到最後子問題可以簡單的直接求解，原問題的解即子問題的解的合併。

合併排序法大致上可以分為切分與排序
切分:

1. 將原陣列切一半成兩個子陣列
2. 將切好的子陣列再切一半
3. 重複第2步直到所有子陣列只包含一個元素

排序:

1. 排序兩個只剩一個元素並合併為一個陣列
2. 將兩邊排序好的陣列再合併為一個陣列
3. 重複2直到所有陣列合併為一個大陣列

### 合併排序法 參考程式碼

```cpp
void merge_sort (int arr[], int reg[], int start, int end) {
    if (start >= end) return; // 不需要排了，返回 （終止條件）
    int len = end-start, mid = len/2 + start; // 計算長度和中點，中點=(start+end)/2
    int start1 = start, end1 = mid; // 左半的起點和終點
    int start2 = mid+1, end2 = end; // 右半的起點和終點
    merge_sort_recursive(arr, reg, start1, end1); // 排左半邊
    merge_sort_recursive(arr, reg, start2, end2); // 排右半邊
    int k = start; // 設定 k=start，以下為合併部份，如上頁所述 （注意 start=start1)
    while (start1 <= end1 && start2 <= end2)
    reg[k++] = arr[start1] < arr[start2] ? arr[start1++] : arr[start2++];
    while (start1 <= end1) reg[k++] = arr[start1++];
    while (start2 <= end2) reg[k++] = arr[start2++];
    for (k = start; k <= end; k++) arr[k] = reg[k];
}
```

### 合併排序法 複雜度分析

時間複雜度: $ O(n * log_{2}(n)) $

* 兩半都要排序，可以如法泡製，如果再各分兩半，複雜度可以再降低
* 注意合併時的複雜度，不管分幾次，每次比較的總次數都維持在 $ O(n) $
* 這樣一直分下去，到最後只剩一個數，就不用排序了
* 總共分割幾次呢？因每次數量減半，故分割次數約為 $ log_{2}(n) $ 次
* 合併時，每一次的總複雜度都是 $ O(n) $，共有 $ log_{2}(n) $ 次
* 總複雜度為 $ O(n * log_{2}(n)) $

空間複雜度: $O(n)$

假設原未排序的陣列長度為 $n$ ，需要額外的 $n$ 長度的陣列來儲存已排序的陣列，因此空間複雜度為 $O(n)$

## 快速排序法

### 快速排序法 介紹

找到一個基準點(pivot)，將比這個基準點小的元素放至左邊的子陣列，將比這個基準點大的元素放到右邊的子陣列，如此一來便能確定這個基準點的位置會是最終排序過後正確的位置，再對左邊的子陣列與右邊的子陣列重複進行相同的步驟，便能完成排序。

### 快速排序法 參考程式碼

```cpp
void quick_sort(int arr[], int low, int high) {
    if (low < high) {
        int pivot = arr[high]; // 選擇基準值
        int i = low - 1;       // 較小元素的索引位置

        for (int j = low; j < high; j++) {
            if (arr[j] <= pivot) {
                i++;
                int temp = arr[i];
                arr[i] = arr[j];
                arr[j] = temp;
            }
        }
        // 將基準值換到中間正確位置
        int temp = arr[i + 1];
        arr[i + 1] = arr[high];
        arr[high] = temp;
        
        int pi = i + 1; // 基準點的索引

        // 遞迴呼叫
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}
```

### 快速排序法 複雜度分析

時間複雜度: $ O(nlog(n)) $ ~ $ O(n^2) $

最好情況 (Best Case): $O(n \log n)$

* 每次選取的基準值恰好都是該區段的中位數時，陣列會被對半分
* 分割次數: 長度為 $n$ 的陣列可以被分割 $\log_2 n$ 層
* 每層處理時間: 每一層不論如何分割，總共都要掃描 $n$ 個元素來進行比較與移動，耗時 $O(n)$
* 總計: $n \times \log_2 n = O(n \log n)$

最壞情況 (Worst Case): $O(n^2)$

* 當選取的基準值每次都是極大值或極小時（例如對一個已經排好序的陣列進行快速排序，且每次都選最後一個數當 Pivot）
* 分割次數: 陣列不會被對分，而是每次只減少一個元素。這會導致需要進行 $n$ 層的分割
* 每層處理時間: 第一層處理 $n$ 個數，第二層 $n-1$ 個...以此類推。總計: $n + (n-1) + (n-2) + \dots + 1 = \frac{n(n+1)}{2} = O(n^2)$

空間複雜度

最好情況 (Best Case): $O(\log n)$

在理想狀態下，基準點（Pivot）每次都能精準地將陣列對半分，遞迴樹的高度為 $\log n$。每一層遞迴都需要固定的空間來儲存局部變數與回傳位址，因此總空間複雜度為 $O(\log n)$。

最壞情況 (Worst Case): $O(n)$

當輸入陣列已經是排序好的（或完全逆序），且每次都選擇第一個或最後一個元素作為基準點，遞迴樹會退化成一條直線（類似鏈結串列），總共會進行 $n$ 次遞迴。棧空間深度達到 $n$，空間複雜度為 $O(n)$。

平均情況: $O(\log n)$

在隨機選擇基準點或資料分布平均的情況下，遞迴樹的期望高度依然維持在對數級別。

## 基數排序法

### 基數排序法 介紹

備註:報告中及時做網頁中均使用LSD (least significant digit)
一種分配式的排序法(distribution sort)，可以分成LSD或MSD，在不同的情形可以使用不同的方式，使其達到最高效率。在做基數排序法的時候會用到桶子，存放陣列的值。

以LSD為例:

1. 在依據個位數的大小將其分配到0~9的桶子內，並遵守下面的規則:
    1. 數值存放時要從桶子的底部開始
    2. 一旦有其中一個桶子放到第 $n$ 層，其餘桶子在分配時也需要從第 $n$ 層開始放
2. 將剛剛依據個位數放進桶子的數值串接起來，並遵守下面的規則:
    1. 由編號0的桶子取到編號9的桶子
    2. 同一個桶子內，先取放置在底部的
3. 改成對十位數進行分配，重複步驟1、步驟2:
按照以上規則，將陣列的所有數值依據十位數排序
4. 移至百位千位萬位，不斷重複步驟3，直到完成排序

LSD的radix sort適用於位數較少的數列，然而在運算數值較大的陣列則是MSD的radix sort效率會比較高。

### 基數排序法 參考程式碼

```cpp
void radixSort(int arr[], int n) {
    // 1. 取得陣列中的最大值，用來決定需要處理幾個位數
    int max = arr[0];
    for (int i = 1; i < n; i++) {
        if (arr[i] > max) max = arr[i];
    }

    // 準備暫存輸出陣列
    int *output = (int *)malloc(n * sizeof(int));

    // 2. 從個位數開始 (exp = 1, 10, 100...)，直到超過最大值
    for (int exp = 1; max / exp > 0; exp *= 10) {
        int count[10] = {0};

        // --- 計數排序邏輯開始 ---
        
        // 統計當前位數 (exp) 每個數字 (0-9) 出現的次數
        for (int i = 0; i < n; i++) {
            count[(arr[i] / exp) % 10]++;
        }

        // 計算累加次數（這決定了數字在 output 中的最終位置）
        for (int i = 1; i < 10; i++) {
            count[i] += count[i - 1];
        }

        // 從後往前掃描原陣列，根據當前位數填入 output
        // 從後往前是為了確保「穩定性」(Stability)
        for (int i = n - 1; i >= 0; i--) {
            int digit = (arr[i] / exp) % 10;
            output[count[digit] - 1] = arr[i];
            count[digit]--;
        }

        // 將排序好的結果複製回原陣列 arr，準備處理下一個位數
        for (int i = 0; i < n; i++) {
            arr[i] = output[i];
        }
        
        // --- 計數排序邏輯結束 ---
    }

    free(output); // 釋放動態記憶體
}
```

### 基數排序法 複雜度分析

時間複雜度: $O(n) = O(d \cdot (n + k))$

$n$: 待排序元素的個數。
$d$: 最大數字的位數（例如數字 999，位數 $d=3$）。
$k$: 每一位數可能的數值範圍（即基數，十進位下 $k=10$）。

基數排序的核心是由一個外層迴圈套著一個內層的穩定排序（通常是計數排序）

* 外層迴圈: 執行次數等於最大位數 $d$。
* 內層排序 (Counting Sort): 統計頻率: 掃描一遍陣列，耗時 $O(n)$。
  * 累加頻率: 掃描一遍基數陣列（0-9），耗時 $O(k)$。
  * 放置元素: 再掃描一遍陣列，耗時 $O(n)$。
  * 內層總計: $O(n + k)$。

將兩者相乘，我們得到整體的總時間複雜度: $$O(n) = O(d \cdot (n + k))$$

空間複雜度: $O(n + k)$

輔助陣列空間：$O(n)$
在每一輪按位數（如個位、十位、百位）排序時，我們不能直接在原陣列交換（因為會破壞穩定性），必須準備一個長度為 $n$ 的暫存陣列來存放該輪排序後的結果。

桶空間（計數空間）：$O(k)$
這裡的 $k$ 指的是基數（Radix），也就是每一位數可能的取值範圍：如果處理的是十進位數字，每個位數可能是 0-9，則 $k = 10$。如果處理的是英文字母，則 $k = 26$。如果處理的是二進位（位元運算），則 $k = 2$。系統需要開闢 $k$ 個「桶」或一個長度為 $k$ 的計數陣列，用來記錄每個數值出現的次數。

## 學習心得

在實際運用上，感覺 $ O(n^2) $ 的演算法沒有用武之地，只要 $n$ 一變大，整個程式就會卡在那邊。至於快速排序法就比較可惜，剛好遇到最壞情況，不然也不至於那麼慢，遇到這種題目，可能要使用其他的變形(Median-of-Three、Tukey's Ninther)，改善時間複雜度。

整個報告做完，我覺得關於演算的本質就是時間和空間，要不是選擇節省記憶體，放棄速度，不然就是提升速度，但要使用更多的空間。像是基數排序法或是合併排序法，就是利用額外的空間來幫助題序，提升速度。原本課堂上的抽象概念在實作後好像變得更清楚、易懂了。
